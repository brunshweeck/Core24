gbool core::Trace::equals ( Object const & obj) const
overridevirtual

Returns true if the specified object is another Trace instance representing the same execution point as this instance.

Two stack trace a and b are equal if and only if:

a.moduleName().equals(b.moduleName()) &&
a.moduleVersion().equals(b.moduleVersion()) &&
a.className().equals(b.className()) &&
a.methodName().equals(b.methodName()) &&
a.fileName().equals(b.fileName()) &&
a.lineNumber() == b.lineNumber()

◆ toString()

String core::Trace::toString ( ) const
virtual

Returns a string representation of this stack trace element.

@apiNote The format of this string depends on the implementation, but the following examples may be regarded as typical:

  • "foo@9.0/com::foo::Main.run(Main.cpp:101)"
    • See the description below.
  • "com.foo.loader/foo@9.0/com::foo::Main.run(Main.cpp)"
    • The line number is unavailable.
  • "foo@9.0/com.foo.Main.run(?)"
    • Neither the file name nor the line number is available.
  • "foo@9.0/com::foo::Main.run(#)"
    • The method containing the execution point is a native method.
  • "com::foo::bar::App.run(App.cpp:12)"
    • The class of the execution point is defined in the unnamed module.
  • "acme@2.1/org::acme.Lib.test(Lib.cpp:80)"
    • The class of the execution point is defined in acme module.
  • "MyClass.mash(MyClass.cpp:9)"
    • MyClass class is on the application class path.

The first example shows a stack trace element consisting of three elements, each separated by "/", followed by the source file name and the line number of the source line containing the execution point.

The first element "foo@9.0" is the module name and version. The seconds element is the method containing the execution point; "com.foo.Main"" is the fully-qualified class name and "run" is the name of the method. @c "Main.cpp" is the source file name and @c "101" is the line number. If a class is defined in an <em>unnamed module</em> then the second element is omitted as shown in @c "com::foo::bar::App.run(App.cpp:12)". If the module is an unnamed module, the second element and its following @c "/" are also omitted as shown in @c "MyClass.mash(MyClass.cpp:9)".

The toString method may return two different values on two Trace instances that are equal, for example one created via the constructor, and one obtained from Throwable, where an implementation may choose to omit some element in the returned string.
