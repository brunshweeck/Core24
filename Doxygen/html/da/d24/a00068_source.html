30 * Indicates whether some other object is "equal to" this one.
31 *
32 * The @c equals method implements an equivalence relation
33 * on non-null object references:
34 * @li It is @a reflexive : For any reference value @c x,
35 * @c x.equals(x) should return @c true.
36 * @li It is @a symetric : For any reference values @c x and @c y,
37 * @c x.equals(y) should return @c true if and only if @c y.equals(x)
38 * return @c true.
39 * @li It is @a transitive : For any reference values @c x, @c y and @c z,
40 * if @c x.equals(y) return @c true, and @c y.equals(z) return @c true,
41 * then @c x.equals(z) should be return @c true
42 * @li It is @a consistant : For any reference value @c x and @c y,
43 * mutliple invocation of @c x.equals(y) consistently return @c true,
44 * or consistently return @c false provided no information used in
45 * @c equals comparisons on the objects is modified.
46 *
47 * @param obj The object to be compared
48 * @return @c true if object instance equals to given object.
49 */

52 /**
53 * Returns a hash code value for the object. This method is
54 * supported for the benefit of hash tables such as those provided by
55 * @c core::util::HashTable.
56 *
57 * The general contract of @c hash is:
58 * @li Whenever it is invoked on the same object more than once during
59 * an execution of a application, the @c hash method
60 * must consistently return the same integer, provided no information
61 * used in @c equals comparisons on the object is modified.
62 * This integer need not remain consistent from one execution of an
63 * application to another execution of the same application.
64 * @li If two objects are equal according to the @c equals
65 * method, then calling the @c hash
66 * method on each of the two objects must produce the
67 * same integer result.
68 * @li It is @a not required that if two objects are unequal
69 * according to the @c equals method, then calling the
70 * @c hash method on each of the two objects must produce
71 * distinct integer results. However, the programmer
72 * should be aware that producing distinct integer results for
73 * unequal objects may improve the performance of hash tables.
74 *
75 * @note
76 * As far as is reasonably practical, the @c hash method defined
77 * by class @c Object returns distinct integers for distinct objects.
78 *
79 * @return The hash code value for this object
80 */
