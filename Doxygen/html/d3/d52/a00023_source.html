12 /**
13 * The class @c Comparable is defined to impose total ordering
14 * on the objects of each class that implement it.
15 * This ordering is referred to as the class's @a natural @a ordering,
16 * and class's @c Comparable::compareTo method refere to as it @a natural
17 * @a ordering.
18 *
19 * Lists (and arrays) of objects that implements this interface can be sorted
20 * automatically by @c Collection::sort method (and @c Arrays::sort method).
21 * Objects that implement this interface can be used as keys in the @a sorted
22 * @a maps or as element in the @a sorted @a sets, without the need to specify
23 * the @a comparator.
24 *
25 * The natural ordering for a class @c C is said to be @a consistent @a with @a equals
26 * if and only if @c x.compareTo(y) == @c 0 has same boolean value as @c x.equals(y)
27 * for every @c x and @x y, instance of class @c C.
28 *
29 * It is strongly recommended (though not required) that natural orderings be
30 * consistent with equals. This is so because sorted sets (and sorted maps)
31 * without explicit comparators behave "strangely" when they are used with
32 * elements (or keys) whose natural ordering is inconsistent with equals. In
33 * particular, such a sorted setValue (or sorted map) violates the general contract
34 * for setValue (or map), which is defined in terms of the @c equals method.
35 *
36 * For example, if one adds two keys @c a and @c b such that
37 * @code !a.equals(b) && (a.compareTo(b) == 0) @endcode to a sorted set
38 * that does not use an explicit comparator, the second @c add operation
39 * return false (and the size of the sorted setValue does not increase)
40 * because @c a and @c b are equivalent from the sorted set's perspective.
41 *
42 * For the mathematically inclined, the @a relation that defines
43 * the natural ordering on a given class C is:
44 * for all @c x and @c y, such that @c x.compareTo(y) <= @c 0.
45 * The @a quotient for this total order is:
46 * for all @c x and @c y, such that @c x.compareTo(y) == @c 0.
47 *
48 * @tparam T The type of objects that this object may be compared to.
49 */
50 template <class T>
51 class Comparable : public virtual Object
52 {
53 public:
54 /**
55 * Compares this object with the specified object for order.
56 * Returns negative integer, zero, or a positive integer as
57 * this object is less than, equal to, or greater than the
58 * specified object.
59 *
60 * The implementor must ensure @a signum:
61 * @c signum(x.compareTo(y)) == @c -signum(y.compare(x))
62 * for every @c x and @c y. This implies that @c x.compareTo(y)
63 * throw an exception if and only if @c y.compareTo(x)
64 * throw an exception.
65 *
66 * The implementor must also ensure that the relation is transitive:
67 * @c (x.compareTo(y) > 0) && @c (y.compareTo(z) > 0) implies
68 * @c (x.compareTo(z) > 0).
69 *
70 * Finally, the implementor must ensure that @c x.compareTo(y) == 0
71 * implies that @c signum(x.compareTo(z)) == @c signum(y.compareTo(z)),
72 * for all @c z.
73 *
74 * @note It is strongly recommended, but @a not strictly required that
75 * @c (x.compareTo(y)==0) == @c (x.equals(y)). Generally speaking, any
76 * class that implements the @c Comparable class and violates
77 * this condition should clearly indicate this fact. The recommended
78 * language is "Note: this class has a natural ordering that is
79 * inconsistent with equals."
80 *
81 * @param o The object to be compared
82 * @return a negative integer, zero, or a positive integer as this object
83 * is less than, equal to, or greater than the specified object.
84 */
85 virtual gint compareTo(T const& o) const = 0;
86 };
