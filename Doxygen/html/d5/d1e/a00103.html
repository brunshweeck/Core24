Detailed Description

The Byte class wraps a value of primitive type gbyte in an object.

An object of type Byte contains a single field whose type is gbyte.

static gbyte core::Byte::parseByte ( String const & s)
static

Parses the string argument as a signed decimal byte.

The characters in the string must all be decimal digits, except that the first character may be an ASCII minus sign '-'('&#92;u002D') to indicate a negative value or an ASCII plus sign '+'('&#92;u002B') to indicate a positive value. The resulting gbyte value is returned, exactly as if the argument and the radix 10 were given as arguments to the parseByte(String,gint) method.

◆ parseByte() [2/2]

static gbyte core::Byte::parseByte ( String const & s,
gint radix )
static

Parses the string argument as a signed gbyte in the radix specified by the second argument.

The characters in the string must all be digits, of the specified radix (as determined by whether Character::digit(gchar,gint) returns a non-negative value) except that the first character may be an ASCII minus sign '-'('&#92;u002D') to indicate a negative value or an ASCII plus sign '+'('&#92;u002B') to indicate a positive value. The resulting gbyte value is returned.

An exception of type NumberFormatException is thrown if any of the following situations occurs:

  • The first argument is a string of length zero.
  • The radix is either smaller than Character::MIN_RADIX or larger than Character::MAX_RADIX.
  • Any character of the string is not a digit of the specified radix, except that the first character may be a minus sign '-'('&#92;u002D') or plus sign '+'('&#92;u002B') provided that the string is longer than length 1.
  • The value represented by the string is not a value of type gbyte.

Returns
the gbyte value represented by the string argument in the specified radix
Exceptions
NumberFormatExceptionIf the string does not contain a parsable gbyte.

◆ toUnsignedInt()

static gint core::Byte::toUnsignedInt ( gbyte x)
static

Converts the argument to an gint by an unsigned conversion.

In an unsigned conversion to an gint, the high-order 24 bits of the gint are zero and the low-order 8 bits are equal to the bits of the gbyte argument.

Consequently, zero and positive gbyte values are mapped to a numerically equal gint value and negative gbyte values are mapped to an gint value equal to the input plus 2**8.

Returns a Byte object holding the value given by the specified String.

The argument is interpreted as representing a signed decimal gbyte, exactly as if the argument were given to the parseByte(String) method. The result is a Byte object that represents the gbyte value specified by the string.

In other words, this method returns a Byte object equal to the value of: Byte::valueOf(Byte::parseByte(s)).
